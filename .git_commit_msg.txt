Rework Overview page per feedback: drop Status widget, add standalone page

The native Status > Overview widget slot from the previous commit was tried
and explicitly rejected by the user ("не надо туда его") - remove it entirely
(files/www/luci-static/resources/view/status/include/60_passwall2-presets.js).

In its place: a standalone chrome-less duplicate page at
admin/services/passwall2-presets/widget, hidden from the menu tree (no
title), backed by a new ucode template
(files/usr/share/ucode/luci/template/passwall2-presets/widget.ut) that uses
luci-theme-bootstrap's own native "blank_page" flag - the same mechanism
sysauth.ut and luci-app-commands' commands_public.ut use - to drop the
navbar/sidebar/footer chrome. It instantiates the exact same
passwall2-presets/overview view, so the admin tab and the standalone page
share one implementation.

overview.js changes:
- Badges now use only native luci-theme-bootstrap classes/CSS variables:
  .label.success (green/working), .label.warning (yellow/transitional -
  degraded, down-in-grace, restarting, restarted-confirming), bare .label
  (grey/undetermined/disabled), and .label with an inline style reusing the
  theme's own --error-color-high/--on-error-color vars for red (the theme has
  no .label.danger modifier, so this reuses its existing error tokens instead
  of inventing a new color).
- Recent events section is now hidden by default behind a checkbox; the
  underlying data keeps refreshing regardless so it's current when revealed.
- poll interval made explicit: poll.add(fn, 5) (5 seconds), per request that
  other status fields refresh at least that often.

Docs updated: SPEC_v0.6.0.md §4 (Overview page) and §5 item 6, BACKLOG.md P7
"still open" list - all record the widget removal/replacement and the
finalized badge scheme.
